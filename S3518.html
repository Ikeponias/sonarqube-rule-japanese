除算やモジュロ演算の分母がゼロの場合、致命的なエラーとなる。

double やfloat で作業する場合、致命的なエラーは発生しませんが、異常な結果につながるため、いずれにせよ避けるべきです。

このルールは、プリミティブなint,long,double,float に加え、BigDecimal とBigInteger をサポートします。

コンプライアンス違反のコード例

void test_divide() {
  int z = 0;
  if (unknown()) {
    // ..
    z = 3;
  } else {
    // ..
  }
  z = 1 / z; // Noncompliant, possible division by zero
}

コンプライアンス・ソリューション

void test_divide() {
  int z = 0;
  if (unknown()) {
    // ..
    z = 3;
  } else {
    // ..
    z = 1;
  }
  z = 1 / z;
}
