assert なぜなら、アサーションはJVMの実行時に無効にすることができ、操作設定が悪いと意図したチェックを完全に排除してしまうからです。さらに、失敗したassertは、何らかのException を投げるのではなく、AssertionErrorを投げる。Errorを投げることは、通常のプログラムにおいて期待されるcatch/throw の動作から完全に外れている。

このルールは、public のメソッドが、1つ以上のパラメータをassertの s で使用する場合に問題となる。

コンプライアンス違反のコード例

 public void setPrice(int price) {
  assert price >= 0 && price <= MAX_PRICE;
  // Set the price
 }

コンプライアンス・ソリューション

 public void setPrice(int price) {
  if (price < 0 || price > MAX_PRICE) {
    throw new IllegalArgumentException("Invalid price: " + price);
  }
  // Set the price
 }
