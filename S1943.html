デフォルトのシステムエンコーディングに依存するクラスやメソッドを使用すると、「ホーム」環境では問題なく動作するコードを作成できます。しかし、そのコードは、異なるエンコーディングを使用している顧客にとっては、診断が非常に困難で、修正するときに再現することがほぼ不可能な方法で壊れるかもしれません。

このルールは、以下のクラスとメソッドの使用を検出します：

  • FileReader
  • FileWriter
  • byte[] の引数を持ち、Charset の引数を持たない文字列コンストラクタ。
    • String(byte[] bytes)
    • String(byte[] bytes, int offset, int length)
  • String.getBytes()
  • String.getBytes(int srcBegin, int srcEnd, byte[] dst, int dstBegin)
  • InputStreamReader(InputStream in)
  • OutputStreamWriter(OutputStream out)
  • ByteArrayOutputStream.toString()
  • 一部Formatter のコンストラクタ
    • Formatter(String fileName)
    • Formatter(File file)
    • Formatter(OutputStream os)
  • いくつかのScanner コンストラクタ
    • Scanner(File source)
    • Scanner(Path source)
    • Scanner(InputStream source)
  • 一部PrintStream のコンストラクタ
    • PrintStream(File file)
    • PrintStream(OutputStream out)
    • PrintStream(OutputStream out, boolean autoFlush)
    • PrintStream(String fileName)
  • いくつかのPrintWriter コンストラクタ
    • PrintWriter(File file)
    • PrintWriter(OutputStream out)
    • PrintWriter(OutputStream out, boolean autoFlush)
    • PrintWriter(String fileName)
  • Apache commons-io ライブラリのメソッドで、エンコーディングの引数が NULL のときに受け付けるもの、およびエンコーディングの引数を省略したこれらのメソッドのオーバーロード。
    • IOUtils.copy(InputStream, Writer)
    • IOUtils.copy(Reader, OutputStream)
    • IOUtils.readLines(InputStream)
    • IOUtils.toByteArray(Reader)
    • IOUtils.toByteArray(String)
    • IOUtils.toCharArray(InputStream)
    • IOUtils.toInputStream(TypeCriteria.subtypeOf(CharSequence))
    • IOUtils.toString(byte[])
    • IOUtils.toString(URI)
    • IOUtils.toString(URL)
    • IOUtils.write(char[], OutputStream)
    • IOUtils.write(CharSequence, OutputStream)
    • IOUtils.writeLines(Collection, String, OutputStream)
    • FileUtils.readFileToString(File)
    • FileUtils.readLines(File)
    • FileUtils.write(File, CharSequence)
    • FileUtils.write(File, CharSequence, boolean)
    • FileUtils.writeStringToFile(File, String)
