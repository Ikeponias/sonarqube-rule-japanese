Java 8以前は、Javaでクロージャを部分的にサポートする唯一の方法は、匿名のインナークラスを使用することでした。しかし、匿名クラスの構文は扱いにくく、不明瞭に感じられるかもしれません。

Java 8では、ソースコードの可読性を高めるために、匿名内部クラスのほとんどの使用をラムダで置き換える必要があります。

なお、このルールは、プロジェクトのsonar.java.source が8 よりも低い場合、自動的に無効となります。

コンプライアンス違反のコード例

myCollection.stream().map(new Mapper<String,String>() {
  public String map(String input) {
    return new StringBuilder(input).reverse().toString();
  }
});

Predicate<String> isEmpty = new Predicate<String> {
    boolean test(String myString) {
        return myString.isEmpty();
    }
}

コンプライアンス・ソリューション

myCollection.stream().map(input -> new StringBuilder(input).reverse().toString());

Predicate<String> isEmpty = myString -> myString.isEmpty();
