WebViewは、モバイルアプリケーションの一部として、Webコンテンツを表示するために使用することができます。コンテンツのレンダリングと表示には、ブラウザエンジンが使用されます。WebViewを使用するモバイルアプリケーションは、Webアプリケーションと同様に、信頼できないコードが表示された場合、クロスサイトスクリプティングの脆弱性を持つ可能性があります。WebViewのコンテキストでは、JavaScriptコードが機密性の高いローカルファイルを流出させたり、さらに悪いことに、アプリケーションの公開関数にアクセスして、コードインジェクションなどのより深刻な脆弱性を引き起こす可能性があります。したがって、絶対に必要であり、Webリソースの信頼性が保証される場合を除き、WebViewでJavaScriptのサポートを有効にすべきではありません。

ということを自問自答してください。

  • WebWiewは、JavaScriptコードの実行を必要としない静的なWebコンテンツのみを描画します。
  • WebViewには信頼できないデータが含まれており、レンダリング時に害を及ぼす可能性があります。

これらの質問のいずれかに「はい」と答えた方はリスクがあります。

推奨されるセキュアコーディングの慣行

JavaScriptコードを実行する必要がない限り、WebViewのJavaScriptサポートを無効にすることが推奨されます。信頼できるページのみをレンダリングする必要があります。

センシティブコードの例

import android.webkit.WebView;

WebView webView = (WebView) findViewById(R.id.webview);
webView.getSettings().setJavaScriptEnabled(true); // Sensitive

コンプライアンス・ソリューション

import android.webkit.WebView;

WebView webView = (WebView) findViewById(R.id.webview);
webView.getSettings().setJavaScriptEnabled(false);
