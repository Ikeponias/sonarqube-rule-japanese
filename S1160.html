チェックされた例外を使用すると、メソッド呼び出し元はエラーを伝播させるか、処理することによって、エラーに対処することを余儀なくされます。例外を投げることで、例外は完全にメソッドのAPIの一部となります。

しかし、呼び出し側の複雑さを合理的に保つために、メソッドは複数の種類のチェックされた例外を投げないようにする必要があります。

コンプライアンス違反のコード例

public void delete() throws IOException, SQLException {      // Noncompliant
  /* ... */
}

コンプライアンス・ソリューション

public void delete() throws SomeApplicationLevelException {
  /* ... */
}

例外あり

オーバーライドメソッドはこのルールではチェックされず、チェックされたいくつかの例外を投げることが許可されます。
