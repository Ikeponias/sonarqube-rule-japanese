命名規則を共有することで、チームは効果的にコラボレーションを行うことができます。このルールでは、ローカル変数や関数のパラメータ名が指定された正規表現に一致しない場合に問題が発生します。

コンプライアンス違反のコード例

デフォルトの正規表現で^[a-z][a-zA-Z0-9]*$ ：

public void doSomething(int my_param) {
  int LOCAL;
  ...
}

コンプライアンス・ソリューション

public void doSomething(int myParam) {
  int local;
  ...
}

例外あり

ループカウンターはこのルールでは無視されます。

for (int i_1 = 0; i_1 < limit; i_1++) {  // Compliant
  // ...
}

のほか、1文字のcatch 変数があります：

try {
//...
} catch (Exception e) { // Compliant
}
