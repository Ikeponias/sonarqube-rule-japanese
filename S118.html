いくつかの命名規則を共有することは、チームが効率的にコラボレーションを行うための重要なポイントです。このルールでは、abstract のクラス名が提供された正規表現に一致するかどうかをチェックすることができます。抽象化されていないクラスが正規表現に一致した場合、そのクラスを抽象化するか、名前を変更するかを提案する問題が提起されます。

コンプライアンス違反のコード例

デフォルトの正規表現では、^Abstract[A-Z][a-zA-Z0-9]*$ ：

abstract class MyClass { // Noncompliant
}

class AbstractLikeClass { // Noncompliant
}

コンプライアンス・ソリューション

abstract class AbstractClass {
}

class LikeClass {
}
