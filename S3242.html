再利用性を最大限に高めるために、メソッドはできるだけ特殊化せずにパラメータを受け取る必要があります。したがって、子クラスの特定の機能がメソッドに要求されない限り、クラス階層の上位にある型を代わりに使用する必要があります。

コンプライアンス違反のコード例

public void printSize(ArrayList<Object> list) {  // Collection can be used instead
    System.out.println(list.size());
}

public static void loop(List<Object> list) { // java.lang.Iterable can be used instead
   for (Object o : list) {
     o.toString();
  }
}

コンプライアンス・ソリューション

public void printSize(Collection<?> list) {  // Collection can be used instead
    System.out.println(list.size());
}

public static void loop(Iterable<?> list) { // java.lang.Iterable can be used instead
   for (Object o : list) {
     o.toString();
  }
}

例外あり

非公開メソッドのパラメータはチェックされません。なぜなら、そのようなメソッドは一般的に再利用されることを意図していないからです。java.lang.String 文字列は不変であり、より汎用的な型に常に置き換えることができないため、パラメータは除外されます。また、メソッドの呼び出しやループの強化以外のコンテキストで使用されるパラメータも除外されます。
