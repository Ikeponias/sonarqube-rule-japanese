getClassfinal なぜなら、子クラスは親や互いに異なるオブジェクトで同期するため、 キーワードにもかかわらず、コードブロックに複数のスレッドを一度に入れることができるためです。synchronized

代わりに、同期させるクラスの名前をハードコードするか、そのクラスをfinal にします。

コンプライアンス違反のコード例

public class MyClass {
  public void doSomethingSynchronized(){
    synchronized (this.getClass()) {  // Noncompliant
      // ...
    }
  }

コンプライアンス・ソリューション

public class MyClass {
  public void doSomethingSynchronized(){
    synchronized (MyClass.class) {
      // ...
    }
  }
