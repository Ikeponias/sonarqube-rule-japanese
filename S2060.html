Externalizable クラスは、それ自身のSerialization とデシリアライズを処理するものである。デシリアライズの際、プロセスの最初のステップは、クラスの引数なしコンストラクタを使用したデフォルトのインスタンス化です。したがって、引数なしコンストラクタを持たないExternalizable クラスは、デシリアライズすることができません。

コンプライアンス違反のコード例

public class Tomato implements Externalizable {  // Noncompliant; no no-arg constructor

  public Tomato (String color, int weight) { ... }
}

コンプライアンス・ソリューション

public class Tomato implements Externalizable {

  public Tomato() { ... }
  public Tomato (String color, int weight) { ... }
}
