ラップされたプリミティブ値（例：Integers とFloats ）を三項演算子（例：a?b:c ）で使用する場合、両方の値はボックスから外れて共通の型に強制されるため、予期せぬ結果につながる可能性があります。これを避けるには、互換性のある型への明示的なキャストを追加します。

コンプライアンス違反のコード例

Integer i = 123456789;
Float f = 1.0f;
Number n = condition ? i : f;  // Noncompliant; i is coerced to float. n = 1.23456792E8

コンプライアンス・ソリューション

Integer i = 123456789;
Float f = 1.0f;
Number n = condition ? (Number) i : f;  // n = 123456789
