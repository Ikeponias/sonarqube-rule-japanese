NullPointerException は避けるべきであり、捕らえるべきではありません。 が明示的にキャッチされるような状況は、簡単に テストに変換できます。また、キャッチブロック内で実行される動作は、簡単に条件分岐の "is null" に移動させることができます。NullPointerException null

コンプライアンス違反のコード例

public int lengthPlus(String str) {
  int len = 2;
  try {
    len += str.length();
  }
  catch (NullPointerException e) {
    log.info("argument was null");
  }
  return len;
}

コンプライアンス・ソリューション

public int lengthPlus(String str) {
  int len = 2;

  if (str != null) {
    len += str.length();
  }
  else {
    log.info("argument was null");
  }
  return len;
}
