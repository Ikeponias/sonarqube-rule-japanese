Iterator.hasNext() を呼び出すと、副作用が発生しないため、イテレータの状態が変化しないはずです。Iterator.next() はイテレータを1つ進めます。そのため、Iterator.hasNext() の内部でこれを呼び出すと、hasNext() の契約が破棄され、実運用では予期せぬ動作につながることになります。

コンプライアンス違反のコード例

public class FibonacciIterator implements Iterator<Integer>{
...
@Override
public boolean hasNext() {
  if(next() != null) {
    return true;
  }
  return false;
}
...
}
