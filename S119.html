命名規則を共有することで、チームでの共同作業を効率的に行うことができます。型パラメーターの名前を1文字にするという確立された慣例に従うことで、コードのユーザーやメンテナンス担当者は、型パラメーターと不適切な名前のクラスとの違いをすぐに見分けることができます。

このルールは、すべての型パラメーターの名前が指定された正規表現に一致するかどうかをチェックします。以下のコードでは、デフォルトの正規表現が使用されています。

コンプライアンス違反のコード例

public class MyClass<TYPE> { // Noncompliant
  <TYPE> void method(TYPE t) { // Noncompliant
  }
}

コンプライアンス・ソリューション

public class MyClass<T> {
  <T> void method(T t) {
  }
}
