sealed クラスは、Java 17で導入されました。この機能は、厳密な階層を定義し、クラスを拡張する可能性を制限する必要がある場合に、非常に便利です。許可されたすべてのサブクラスについて言及するために、キーワード があり、これにサブクラスの名前を続ける必要があります。permits

この表記は、あるsealed クラスのサブクラスが、異なるファイル、パッケージ、あるいはモジュールに存在する場合に非常に便利です。すべてのサブクラスが同じファイルで宣言されている場合、明示的に記述する必要はなく、宣言のpermits の部分を省略することができます。

このルールは、sealed クラスのすべてのサブクラスが、そのスーパークラスと同じファイルで宣言されている場合の問題を報告します。

コンプライアンス違反のコード例

sealed class A permits B, C, D, E {} // Noncompliant
final class B extends A {}
final class C extends A {}
final class D extends A {}
final class E extends A {}

コンプライアンス・ソリューション

sealed class A {} // Compliant
final class B extends A {}
final class C extends A {}
final class D extends A {}
final class E extends A {}
