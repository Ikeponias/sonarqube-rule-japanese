java.util.function パッケージは、ラムダ式やメソッド参照で使用する機能的なインターフェイスの定義を多数提供しています。一般に、オートボックス化を避けるために、より特殊な形式を使用することが推奨されます。例えば、IntFunction<Foo>はFunction<Integer, Foo> よりも優先されるべきです。

このルールは、以下のいずれかの置換が可能な場合に問題となる：

コンプライアンス違反のコード例

public class Foo implements Supplier<Integer> {  // Noncompliant
    @Override
    public Integer get() {
      // ...
    }
}

public class Foo implements IntSupplier {

  @Override

  }
