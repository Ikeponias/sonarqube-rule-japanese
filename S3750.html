Spring@Controller,@Service,@Repositoryは、デフォルトでsingleton のスコープを持っています。つまり、アプリケーションでインスタンス化されるのは、このクラスの1インスタンスだけです。これらのクラスタイプに他のスコープを定義すると、新しいインスタンスが作成されたり破棄されたりして、無駄な動作が発生することになります。多忙なWebアプリケーションでは、サーバーに無用な追加負荷がかかる可能性があります。

このルールは、@Scope のアノテーションが、@Controller 、@Service 、@Repository のうち、「singleton」以外の値に適用された場合に問題となります。@Scope("singleton") は冗長ですが、無視します。

@Scope("prototype")  // Noncompliant
@Controller
public class HelloWorld {
