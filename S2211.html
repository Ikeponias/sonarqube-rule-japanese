コーディング規約を共有することで、チームは効果的にコラボレーションを行うことができます。ラムダ引数の型は任意ですが、とにかく指定することで、コードがより明確になり、読みやすくなります。

コンプライアンス違反のコード例

Arrays.sort(rosterAsArray,
    (a, b) -> {  // Noncompliant
        return a.getBirthday().compareTo(b.getBirthday());
    }
);

コンプライアンス・ソリューション

Arrays.sort(rosterAsArray,
    (Person a, Person b) -> {
        return a.getBirthday().compareTo(b.getBirthday());
    }
);

例外あり

ラムダが1つか2つのパラメータを持ち、ブロックを持たない場合、このルールは問題を起こさないでしょう。

stream.map((a, b) -> a.length); // compliant
