最終的なdefault 節の要件は、防御的なプログラミングです。この節は、適切な行動をとるか、なぜ行動をとらないかについての適切なコメントを含む必要があります。

コンプライアンス違反のコード例

switch (param) {  //missing default clause
  case 0:
    doSomething();
    break;
  case 1:
    doSomethingElse();
    break;
}

switch (param) {
  default: // default clause should be the last one
    error();
    break;
  case 0:
    doSomething();
    break;
  case 1:
    doSomethingElse();
    break;
}

コンプライアンス・ソリューション

switch (param) {
  case 0:
    doSomething();
    break;
  case 1:
    doSomethingElse();
    break;
  default:
    error();
    break;
}

例外あり

switch パラメータがEnum であり、この enum のすべての定数がcase ステートメントで使用される場合、default 節は期待されない。

例

public enum Day {
    SUNDAY, MONDAY
}
...
switch(day) {
  case SUNDAY:
    doSomething();
    break;
  case MONDAY:
    doSomethingElse();
    break;
}

見る

  • MITRE, CWE-478- Switchステートメントにデフォルトケースがない。
  • CERT、MSC01-C。- 論理的な完全性を追求する
