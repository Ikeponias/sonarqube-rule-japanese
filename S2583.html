常にtrue やfalse となるような条件式は、デッドコードになりかねません。このようなコードは常にバギーであり、実運用では決して使用してはいけません。

コンプライアンス違反のコード例

a = false;
if (a) { // Noncompliant
  doSomething(); // never executed
}

if (!a || b) { // Noncompliant; "!a" is always "true", "b" is never evaluated
  doSomething();
} else {
  doSomethingElse(); // never executed
}

例外あり

このルールは、どちらのケースでも問題になることはありません：

  • 条件がシングルfinal boolean の場合 
final boolean debug = false;
//...
if (debug) {
  // Print something
}
  • 文字通り、true 、false の状態のとき。 
if (true) {
  // do something
}

このような場合、コードが意図したとおりになっていることは明らかです。

  • MITRE, CWE-571- 式は常に真である。
