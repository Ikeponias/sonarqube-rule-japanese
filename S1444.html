フィールドを "final "と宣言せずに "public "と "static "と宣言する正当な理由はない。ほとんどの場合、これは複数のオブジェクトの間で状態を共有するためのごまかしです。しかし、この方法では、どのオブジェクトも、共有された状態に対して、null に設定するなど、好きなことをすることができます。

コンプライアンス違反のコード例

public class Greeter {
  public static Foo foo = new Foo();
  ...
}

コンプライアンス・ソリューション

public class Greeter {
  public static final Foo FOO = new Foo();
  ...
}
