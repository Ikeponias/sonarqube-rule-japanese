Date とCalendar のクラスは、特にマルチスレッドコンテキストでは、常に混乱し、適切に使用するのが困難でした。JodaTime は、長い間人気のある代替手段でしたが、今ではさらに優れたオプションが組み込まれています。Java 8のJSR 310実装では、以下のような特定のクラスを提供しています：

クラス に使用する。

ローカルデイト

日付（時刻、オフセット、ゾーンなし）。

ローカルタイム

日付、オフセット、ゾーンを除いた、その日の時刻を表示します。

ローカルデイトタイム

オフセットなしの日付と時刻、またはゾーンを表示します。

オフセット日

02:00のようなオフセットを持つ日付で、時刻やゾーンは含まれません。

オフセットタイム

02:00のようなオフセットを持つ時刻を、日付やゾーンなしで表示します。

オフセットデートタイム

日付と時刻に+02:00のようなオフセットがあり、ゾーンがない場合

ゾーンドデイトタイム

日付と時刻、タイムゾーンとオフセットを表示します。

コンプライアンス違反のコード例

Date now = new Date();  // Noncompliant
DateFormat df = new SimpleDateFormat("dd.MM.yyyy");
Calendar christmas  = Calendar.getInstance();  // Noncompliant
christmas.setTime(df.parse("25.12.2020"));

コンプライアンス・ソリューション

LocalDate now = LocalDate.now();  // gets calendar date. no time component
LocalTime now2 = LocalTime.now(); // gets current time. no date component
LocalDate christmas = LocalDate.of(2020,12,25);
