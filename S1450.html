プライベートフィールドの値が、クラスのメソッドで常に代入されてから読み込まれる場合、クラスの情報を保存するために使用されていないことになります。したがって、誤解を防ぐために、関連するメソッドではローカル変数にする必要があります。

コンプライアンス違反のコード例

public class Foo {
  private int a;
  private int b;

  public void doSomething(int y) {
    a = y + 5;
    ...
    if(a == 0) {
      ...
    }
    ...
  }

  public void doSomethingElse(int y) {
    b = y + 3;
    ...
  }
}

コンプライアンス・ソリューション

public class Foo {

  public void doSomething(int y) {
    int a = y + 5;
    ...
    if(a == 0) {
      ...
    }
  }

  public void doSomethingElse(int y) {
    int b = y + 3;
    ...
  }
}

例外あり

このルールは、アノテーションフィールドに関する問題を生じさせません。
