インフラストラクチャの問題などでテストが失敗した場合、一時的に無視したいことがあるかもしれません。しかし、なぜそのテストが無視されるのかについて何らかの表記がなければ、決して再稼働させることはできないかもしれません。このようなテストは、プロジェクトの包括的な知識がないと対処が難しく、結局はそのプロジェクトを汚染してしまいます。

このルールでは、なぜスキップされるのかについてのコメントがない、無視される各テストについて問題が発生します。

  • Junit4では、このルールは@Ignoreアノテーションを対象としています。
  • Junit5では、このルールは@Disabledアノテーションを対象としています。
  • assumeTrue(false)やassumeFalse(true)を使ってテストをスキップしているケースも対象となります。

コンプライアンス違反のコード例

@Ignore  // Noncompliant
@Test
public void testDoTheThing() {
  // ...

または

@Test
public void testDoTheThing() {
  Assume.assumeFalse(true); // Noncompliant
  // ...

コンプライアンス・ソリューション

@Test
@Ignore("See Ticket #1234")
public void testDoTheThing() {
  // ...
