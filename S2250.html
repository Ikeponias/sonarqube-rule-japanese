コレクションに対するメソッド呼び出しの時間的複雑さは、必ずしも明らかではありません。たとえば、ほとんどのコレクションでは、size() メソッドに一定の時間がかかりますが、ConcurrentLinkedQueue.size() を実行するのに必要な時間は O(n)、つまりコレクション内の要素数に正比例します。コレクションが大きい場合、これは高価な操作となる可能性があります。

このルールは、以下のO(n)メソッドがクラスフィールドのコンストラクタ以外で呼び出された場合に問題となる：

  • ArrayList
    • contains
    • remove
  • LinkedList
    • get
    • contains
  • ConcurrentLinkedQueue
    • size
    • contains
  • ConcurrentLinkedDeque
    • size
    • contains
  • CopyOnWriteArrayList
    • add
    • contains
    • remove
  • CopyOnWriteArraySet
    • add
    • contains
    • remove

コンプライアンス違反のコード例

ConcurrentLinkedQueue queue = new ConcurrentLinkedQueue();
//...
log.info("Queue contains " + queue.size() + " elements");  // Noncompliant
