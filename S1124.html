Java Language Specificationでは、修飾語を以下の順序で並べることを推奨しています：

  1. アノテーション
  2. おもてだった
  3. プロテクテッド
  4. 個人
  5. ちゅうしょう
  6. スタティック
  7. 最終
  8. 果敢ない
  9. 揮発性
  10. シンクロナイズド
  11. ネイティブ
  12. デフォルト
  13. 厳密なfp

この規約に従わないことは技術的な影響はありませんが、多くの開発者が標準的な順序に慣れているため、コードの可読性を低下させます。

コンプライアンス違反のコード例

static public void main(String[] args) {   // Noncompliant
}

コンプライアンス・ソリューション

public static void main(String[] args) {   // Compliant
}
