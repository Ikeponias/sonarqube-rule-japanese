メソッドの命名は、tostring 、hashcode 、equal のいずれかです：

  • タイプミスという形のバグです。toString,Object.hashCode() (camelCasingに注意) またはObject.equals (末尾の's'に注意)を上書きすることを意味しており、アプリケーションは期待通りの動作をしない。
  • 意図的に行われたものです。しかし、この名前は他のすべての開発者を混乱させ、名前の違いに気づかないかもしれませんし、バグだと思うかもしれません。

いずれの場合も、メソッドの名称を変更する必要があります。

コンプライアンス違反のコード例

public int hashcode() { /* ... */ }  // Noncompliant

public String tostring() { /* ... */ } // Noncompliant

public boolean equal(Object obj) { /* ... */ }  // Noncompliant

コンプライアンス・ソリューション

@Override
public int hashCode() { /* ... */ }

@Override
public String toString() { /* ... */ }

@Override
public boolean equals(Object obj) { /* ... */ }
