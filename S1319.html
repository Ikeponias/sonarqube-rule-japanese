Java Collections APIの目的は、実装の詳細を隠すために、明確に定義されたインターフェースの階層を提供することである。

新しいコレクションをインスタンス化するには、実装クラスを使用する必要がありますが、インスタンス化の結果は、理想的には、Javaコレクションインタフェースを型とする変数に格納されるべきです。

このルールでは、実装クラスがある場合に問題が発生します：

  • は、public メソッドから返されます。
  • は、public メソッドの引数として受け取ります。
  • は、public メンバーとして公開されています。

コンプライアンス違反のコード例

public class Employees {
  private HashSet<Employee> employees = new HashSet<Employee>();  // Noncompliant - "employees" should have type "Set" rather than "HashSet"

  public HashSet<Employee> getEmployees() {                       // Noncompliant
    return employees;
  }
}

コンプライアンス・ソリューション

public class Employees {
  private Set<Employee> employees = new HashSet<Employee>();      // Compliant

  public Set<Employee> getEmployees() {                           // Compliant
    return employees;
  }
}
