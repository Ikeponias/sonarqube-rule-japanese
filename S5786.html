JUnit5 は、すべてをpublic にする必要があった JUnit4 よりも、Test クラスの可視性に関して寛容です。

そのため、JUnit5 のテストクラスはprivate 以外の可視性を持つことができますが、コードの可読性を高めるために、デフォルトのパッケージ可視性を使用することが推奨されます。

コンプライアンス違反のコード例

import org.junit.jupiter.api.Test;

public class MyClassTest { // Noncompliant - modifier can be removed
  @Test
  protected void test() { // Noncompliant - modifier can be removed
    // ...
  }
}

コンプライアンス・ソリューション

import org.junit.jupiter.api.Test;

class MyClassTest {
  @Test
  void test() {
    // ...
  }
}

例外あり

private テストメソッドやクラスは、JUnit5 で適切な警告が表示されることなく体系的に無視されるため、このルールではprivate の可視性については問題にはなりません。これはCode Smell ではなく、{rule:java:S5810} というルールで処理されるBug です。
