サブクラスのインスタンスはスーパークラスにキャストされ、スーパークラスのインスタンスとして扱われることがあるので、オーバーライドするメソッドは、リスコフ置換原則に関連するスーパークラスの契約の側面を維持する必要があります。具体的には、スーパークラスのメソッドのパラメータや戻り値の型に、以下のいずれかが記されている場合です：@Nullable @CheckForNull 、@NotNull 、@NonNull 、@Nonnull 、サブクラスのパラメータは契約を厳しくすることは許されず、戻り値は契約を緩めることは許されません。

コンプライアンス違反のコード例

public class Fruit {

  private Season ripe;
  private String color;

  public void setRipe(@Nullable Season ripe) {
    this.ripe = ripe;
  }

  public @NotNull Integer getProtein() {
    return 12;
  }
}

public class Raspberry extends Fruit {

  public void setRipe(@NotNull Season ripe) {  // Noncompliant
    this.ripe = ripe;
  }

  public @Nullable Integer getProtein() {  // Noncompliant
    return null;
  }
}
