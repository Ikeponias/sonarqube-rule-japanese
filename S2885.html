標準的なJavaライブラリのすべてのクラスがスレッドセーフであるように書かれているわけではありません。それらをマルチスレッドで使用すると、実行時にデータの問題や例外が発生する可能性が高くなります。

このルールは、Calendar 、DateFormat 、javax.xml.xpath.XPath 、javax.xml.validation.SchemaFactory のインスタンスが、static と表示されている場合に問題となる。

コンプライアンス違反のコード例

public class MyClass {
  private static SimpleDateFormat format = new SimpleDateFormat("HH-mm-ss");  // Noncompliant
  private static Calendar calendar = Calendar.getInstance();  // Noncompliant

コンプライアンス・ソリューション

public class MyClass {
  private SimpleDateFormat format = new SimpleDateFormat("HH-mm-ss");
  private Calendar calendar = Calendar.getInstance();
