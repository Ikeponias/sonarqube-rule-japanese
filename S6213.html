技術的に可能であっても、Restricted Identifiersを識別子として使用するべきではありません。これは互換性のためにのみ可能であり、Javaコードで使用することは混乱を招くので避けるべきです。

なお、将来の混乱を避けるため、これらの識別子がまだ制限されていないバージョンを含む、どのバージョンのJavaにも適用されます。

このルールでは、識別子を制限した場合の問題が報告されています：

  • ヴァル
  • 産する
  • 書き記す

は識別子として使用される。

コンプライアンス違反のコード例

var var = "var"; // Noncompliant: compiles but this code is confusing
var = "what is this?";

int yield(int i) { // Noncompliant
  return switch (i) {
    case 1: yield(0); // This is a yield from switch expression, not a recursive call.
    default: yield(i-1);
  };
}

String record = "record"; // Noncompliant

コンプライアンス・ソリューション

var myVariable = "var";

int minusOne(int i) {
  return switch (i) {
    case 1: yield(0);
    default: yield(i-1);
  };
}

String myRecord = "record";
