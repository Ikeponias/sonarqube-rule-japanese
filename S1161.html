@Override のアノテーションを使用すると、2つの理由で便利です：

  • アノテーションされたメソッドが実際には何もオーバーライドしていない場合、スペルミスのようにコンパイラから警告が表示されます。
  • メソッドがオーバーライドされていることを明白にすることで、ソースコードの可読性を向上させます。

コンプライアンス違反のコード例

class ParentClass {
  public boolean doSomething(){...}
}
class FirstChildClass extends ParentClass {
  public boolean doSomething(){...}  // Noncompliant
}

コンプライアンス・ソリューション

class ParentClass {
  public boolean doSomething(){...}
}
class FirstChildClass extends ParentClass {
  @Override
  public boolean doSomething(){...}  // Compliant
}

例外あり

このルールは、toString(),hashCode(), ...のように、Object クラスのメソッドをオーバーライドするときに緩和されます。
