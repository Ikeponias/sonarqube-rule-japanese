ループ内のbreak とcontinue の文の数を制限することは、優れた構造化プログラミングのために行われます。

最適なコーディングを容易にするため、ループ内ではbreak またはcontinue ステートメントを1つだけ使用することが可能です。複数ある場合は、可読性を高めるためにコードをリファクタリングする必要があります。

コンプライアンス違反のコード例

for (int i = 1; i <= 10; i++) {     // Noncompliant - 2 continue - one might be tempted to add some logic in between
  if (i % 2 == 0) {
    continue;
  }

  if (i % 3 == 0) {
    continue;
  }

  System.out.println("i = " + i);
}
