クラスを静的にインポートすると、クラス名で修飾することなく、そのpublic static メンバーを使用することができます。これは便利ですが、あまりに多くのクラスを静的にインポートすると、コードが混乱し、保守が困難になる可能性があります。

コンプライアンス違反のコード例

デフォルトの閾値で4

import static java.lang.Math.*;
import static java.util.Collections.*;
import static com.myco.corporate.Constants.*;
import static com.myco.division.Constants.*;
import static com.myco.department.Constants.*;  // Noncompliant
