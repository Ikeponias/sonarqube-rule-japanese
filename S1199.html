ネストされたコードブロックは、新しいスコープを作成し、その中で定義された変数の可視性を制限するために使用することができます。この機能をメソッドで使用することは、一般的に、そのメソッドの責務が多すぎることを示し、より小さなメソッドにリファクタリングする必要があります。

コンプライアンス違反のコード例

public void evaluate(int operator) {
  // Do some computation...
  {
    int a = stack.pop();
    int b = stack.pop();
    int result = a + b;
    stack.push(result);
  }
}

コンプライアンス・ソリューション

public void evaluate(int operator) {
  // Do some computation...
  evaluateAdd();
}

private void evaluateAdd() {
  int a = stack.pop();
  int b = stack.pop();
  int result = a + b;
  stack.push(result);
}
