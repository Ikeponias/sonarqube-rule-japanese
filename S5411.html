ボックス型java.lang.Boolean を制御フローを決定する式として使用する場合（Java 言語仕様書 §4.2.5 Theboolean Type and boolean Values に記載）、値がnull の場合はNullPointerException を投げる（Java 言語仕様書 §5.1.8 Unboxing Conversion に記載）。

このような変換は完全に避け、null の値を明示的に処理する方が安全です。

ただし、すでにヌルチェックされているブーリアンについては、問題が発生しないことに注意してください。

コンプライアンス違反のコード例

Boolean b = getBoolean();
if (b) {  // Noncompliant, it will throw NPE when b == null
  foo();
} else {
  bar();
}

コンプライアンス・ソリューション

Boolean b = getBoolean();
if (Boolean.TRUE.equals(b)) {
  foo();
} else {
  bar();  // will be invoked for both b == false and b == null
}


Boolean b = getBoolean();
if(b != null){
  String test = b ? "test" : "";
}
