java.util.concurrent.locks.Lock は、 ブロックで利用可能なものよりはるかに強力で柔軟なロック操作を提供します。そのため、 で同期をとることは、そのオブジェクトの力を捨ててしまうことになり、ただの愚かな行為です。代わりに、このようなオブジェクトは と を使ってロックとアンロックを行う必要があります。synchronized Lock tryLock() unlock()

コンプライアンス違反のコード例

Lock lock = new MyLockImpl();
synchronized(lock) {  // Noncompliant
  //...
}

コンプライアンス・ソリューション

Lock lock = new MyLockImpl();
lock.tryLock();
//...
