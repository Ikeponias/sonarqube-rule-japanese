デッドストアは、ローカル変数に値が代入され、その後の命令では読み取れない場合に発生します。計算や取得した値を上書きしたり捨てたりすることは、コードに重大なエラーがあることを意味します。そのため、計算された値はすべて使用する必要があります。

コンプライアンス違反のコード例

i = a + b; // Noncompliant; calculation result not used before value is overwritten
i = compute();

コンプライアンス・ソリューション

i = a + b;
i += compute();

例外あり

このルールは、-1、0、1、null 、true 、false 、"" の初期化を無視します。

見る

  • MITRE, CWE-563- 使用しない変数（'Unused Variable'）に割り当てる。
  • CERT、MSC13-C。- 未使用値の検出と削除
  • CERT、MSC56-J。- 余計なコードや値を検出して削除する
