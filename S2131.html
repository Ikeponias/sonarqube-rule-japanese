「ボックス化」とは、プリミティブ値をプリミティブ・ラッパー・オブジェクトに入れることです。それが純粋にラッパー・クラスのtoString メソッドを使うために行われる場合、それらのメソッドはstatic 、したがってクラス・インスタンスがなくても使えるので、メモリとサイクルの無駄遣いになっています。同様に、プリミティブ・ラッパークラスのstatic メソッドvalueOf を、String以外の引数で使うことは避けるべきです。

コンプライアンス違反のコード例

int myInt = 4;
String myIntString = (new Integer(myInt)).toString(); // Noncompliant; creates & discards an Integer object
myIntString = Integer.valueOf(myInt).toString(); // Noncompliant

コンプライアンス・ソリューション

int myInt = 4;
String myIntString = Integer.toString(myInt);

非推奨

このルールは非推奨です。代わりに {rule:java:S1158} を使用してください。
