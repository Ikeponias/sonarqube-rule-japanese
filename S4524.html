switch は、様々な理由で 節を含むことができます：予期せぬ値を処理するため、すべてのケースが適切に考慮されたことを示すためです。default

読みやすくするために、switch 文のデフォルトの動作を開発者がすぐに見つけられるように、default 節をswitch 文の最後に置くことが推奨されています。このルールは、default 節がswitch'のケースの最後のものでない場合、問題が発生します。

コンプライアンス違反のコード例

switch (param) {
  case 0:
    doSomething();
    break;
  default: // default clause should be the last one
    error();
    break;
  case 1:
    doSomethingElse();
    break;
}

コンプライアンス・ソリューション

switch (param) {
  case 0:
    doSomething();
    break;
  case 1:
    doSomethingElse();
    break;
  default:
    error();
    break;
}
