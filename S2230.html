@Transactional Springはpublic以外のメソッドを "見ない "ので、そのメソッドを適切に呼び出すための規定を作らないからです。また、Springは呼び出されたメソッドによって呼び出されるメソッドについて規定することもありません。

したがって、private のメソッドを、たとえば@Transactional とマークしても、実行時エラーや例外が発生するのは、そのメソッドが実際には@Transactional となるように書かれている場合だけです。

コンプライアンス違反のコード例

@Transactional  // Noncompliant
private void doTheThing(ArgClass arg) {
  // ...
}
