空の文、すなわち; は、通常、because のように誤って導入されることが多い：

  • 実際の文言に置き換えるつもりだったのですが、これは忘れ去られました。
  • セミコロンが2つになってしまう誤植がありました（;;.

コンプライアンス違反のコード例

void doSomething() {
  ;                                                       // Noncompliant - was used as a kind of TODO marker
}

void doSomethingElse() {
  System.out.println("Hello, world!");;                     // Noncompliant - double ;
  ...
}

コンプライアンス・ソリューション

void doSomething() {}

void doSomethingElse() {
  System.out.println("Hello, world!");
  ...
  for (int i = 0; i < 3; i++) ; // compliant if unique statement of a loop
  ...
}

見る

  • CERT、MSC12-C。- 効果がない、または実行されないコードを検出して削除する
  • CERT、MSC51-J。- if、for、whileの条件の直後にセミコロンを配置しない
  • CERT、EXP15-C。- if、for、while文と同じ行にセミコロンを置かないでください。
