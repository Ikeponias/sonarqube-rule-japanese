慣習として、Javaクラスの物理的な位置（ソースディレクトリ）と論理的な表現（パッケージ）は同期しているはずです。したがって、"src/org/bar/Foo.java" にある Java ファイルは、"org.bar" のパッケージを持つ必要があります。

残念ながら、この規約はJavaコンパイラによって強制されるものではなく、開発者が「Foo.java」クラスを「com.apple」パッケージの一部にすることを防ぐものはありません。このことは、クラスとそのアプリケーションの両方の保守性を低下させる恐れがあります。

同様に、同等のフォルダ構造ではなく、名前にドットを含むフォルダに配置されたソースは、コンパイルはできますが、実行時に問題が発生します。たとえば、パッケージ宣言がorg.foo.bar であるコードをorg/foo.bar に配置すると、コンパイルはできますが、クラスローダーは常にパッケージ構造に基づいてフォルダ内のクラスを検索し、その結果、ソースがorg/foo/bar フォルダにあることを期待します。したがってfoo.bar はソースのフォルダ名として適切ではありません。
