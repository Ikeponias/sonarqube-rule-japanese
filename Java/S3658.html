単体テストでテスト対象のコードが例外を投げる場合、テスト自体が失敗します。したがって、失敗を検出するために、テスト対象のコードをtry-catch の構造で囲む必要はない。その代わりに、例外の種類をメソッドのシグネチャに移すだけでよいのです。

このルールでは、catch ブロックの中に fail アサーションがある場合に問題が発生します。

対応するフレームワーク

  • ジュウニットスリー
  • ジュウニットフォー
  • ジュウニットファイブ
  • フェストアサート
  • アサートジェイ

コンプライアンス違反のコード例

@Test
public void testMethod() {
  try {
            // Some code
  } catch (MyException e) {
    Assert.fail(e.getMessage());  // Noncompliant
  }
}

コンプライアンス・ソリューション

@Test
public void testMethod() throws MyException {
    // Some code
}
