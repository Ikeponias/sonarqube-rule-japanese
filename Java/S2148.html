Java 7から、数値リテラルにアンダースコア（'_'）を付加して可読性を高めることができるようになりました。この方法によるアンダースコアの追加は、意味的な意味はありませんが、保守者がコードを理解しやすくするものです。

このルールを発動するために必要な小数点の左側の桁数は、ベースによって異なります。

ベース 最小桁数

バイナリ

9

八進数

9

10進数

6

十六進法

9

このルールで精査されるのは、アンダースコアの有無だけで、その間隔ではありません。

なお、このルールは、プロジェクトのsonar.java.source が7 よりも低い場合、自動的に無効となります。

コンプライアンス違反のコード例

int i = 10000000;  // Noncompliant; is this 10 million or 100 million?
int  j = 0b01101001010011011110010101011110;  // Noncompliant
long l = 0x7fffffffffffffffL;  // Noncompliant

コンプライアンス・ソリューション

int i = 10_000_000;
int  j = 0b01101001_01001101_11100101_01011110;
long l = 0x7fff_ffff_ffff_ffffL;
