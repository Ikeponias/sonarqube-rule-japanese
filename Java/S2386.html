static ミュータブルなオブジェクトをpublic (デフォルト)、interface のメンバーとして持つ正当な理由はありません。そのような変数はクラスに移し、その可視性を低くするべきです。

同様に、ゲッターやセッターを通してではなく、直接アクセスされるクラスや列挙型のミュータブルメンバー（static ）は、可能な限り保護する必要があります。これは、適切であれば、可視性を下げるか、フィールドをfinal にすることによって行うことができます。

なお、配列のようなミュータブルフィールドをfinal 、変数の再割り当てを行わないようにしますが、そうすることで配列の内部状態のミュータビリティに影響を与えることはありません（つまり、目的を達成することはできません）。

このルールは、public static 配列、Collection 、Date 、awt.Point のメンバーにとって問題となるものです。

コンプライアンス違反のコード例

public interface MyInterface {
  public static String [] strings; // Noncompliant
}

public class A {
  public static String [] strings1 = {"first","second"};  // Noncompliant
  public static String [] strings2 = {"first","second"};  // Noncompliant
  public static List<String> strings3 = new ArrayList<>();  // Noncompliant
  // ...
}
