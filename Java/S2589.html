もし、ブーリアン式が条件の評価を変えないのであれば、全く不要なものであり、削除してよい。プログラマの意図と一致しないためにありがた迷惑であるなら、それはバグであり、その表現は修正されるべきです。

コンプライアンス違反のコード例

a = true;
if (a) { // Noncompliant
  doSomething();
}

if (b && a) { // Noncompliant; "a" is always "true"
  doSomething();
}

if (c || !a) { // Noncompliant; "!a" is always "false"
  doSomething();
}

コンプライアンス・ソリューション

a = true;
if (foo(a)) {
  doSomething();
}

if (b) {
  doSomething();
}

if (c) {
  doSomething();
}
