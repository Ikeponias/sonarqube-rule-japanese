VisibleForTestingを使用すると、メソッド、フィールド、クラスなどの可視性制限がAPIに必要以上に緩和され、ユニットテストが容易にできるようになります。

このようなメソッド、フィールド、クラスへのアクセスは、テストコードでは問題ありませんが、プロダクションコードでは避けなければなりません。プロダクションコードでは、これらのメソッドはprivateであるかのように扱われるべきです。

対応するフレームワーク

  • グアバです： com.google.common.annotations.VisibleForTesting
  • AssertJ:org.assertj.core.util.VisibleForTesting
  • Android：androidx.annotation.VisibleForTesting
  • Apache Flinkです： org.apache.flink.annotation.VisibleForTesting

という名前のアノテーションや、その他のアノテーションがあります。VisibleForTesting

コンプライアンス違反のコード例

/** src/main/java/MyObject.java */

@VisibleForTesting String foo;

/** src/main/java/Service.java */

new MyObject().foo; // Noncompliant, foo is accessed from production code

コンプライアンス・ソリューション

/** src/main/java/MyObject.java */

@VisibleForTesting String foo;

/** src/test/java/MyObjectTest.java */

new MyObject().foo; // Compliant, foo is accessed from test code
