式の複雑さは、それが含む&&,|| およびcondition ? ifTrue : ifFalse演算子の数によって定義されます。

コードの可読性を保つために、1つの式の複雑さが高くなりすぎないようにする。

コンプライアンス違反のコード例

デフォルトの閾値が3である場合：

if (((condition1 && condition2) || (condition3 && condition4)) && condition5) { ... }

コンプライアンス・ソリューション

if ( (myFirstCondition() || mySecondCondition()) && myLastCondition()) { ... }

例外あり

この種のメソッドでは、クラスのすべてのフィールドを等しく比較するのが一般的であるため、equals メソッド内では問題は報告されません。
