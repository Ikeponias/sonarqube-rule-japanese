テストは常に行う必要があります：

  • エッジケースを含め、プロダクションコードが期待通りに動作することを確認する。
  • デバッグが容易であること、すなわち理解しやすく再現性があること。

テストにランダムな値を使うと、必ずしもエッジケースをチェックできないし、テストログも読みづらくなる。読みやすいハードコードされた値を使うほうがよいでしょう。もし、コードが大きくなるようなら、ヘルパー関数を使うことができます。

テストにおけるランダムデータの有効な使用例として、すべての値をテストするとテストが非現実的に遅くなる場合があります。このような場合、長い目で見ればランダムですべての値をテストするのがベストです。しかし、失敗を再現できるように、ランダムな値は必ずログに残すようにしましょう。 これを簡単にするためのライブラリもいくつかあります。例えば、jqwikのようなプロパティベースのテストライブラリを使うことができます。

このルールは、テストコードでnew Random() またはUUID.randomUUID() が呼び出された場合に問題が発生します。

コンプライアンス違反のコード例

int userAge = new Random().nextInt(42);  // Noncompliant
UUID userID = UUID.randomUUID(); // Noncompliant

コンプライアンス・ソリューション

int userAge = 31;
UUID userID = UUID.fromString("00000000-000-0000-0000-000000000001");
