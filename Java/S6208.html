Java 14では、SwitchステートメントとExpressionで、異なるケースに対して同じアクションを実行する場合の新しいケースの書き方があります。 同じアクションを持つ複数のブランチを宣言する代わりに、カンマで区切ってすべてのケースを1つのグループとしてまとめることができます。これにより、より簡潔なコードになり、可読性が向上します。

本ルールでは、本装置の複数のケースをカンマ区切りで1つのケースにグループ化できる場合の問題を報告します。

コンプライアンス違反のコード例

// Switch Expression
int i = switch (mode) {
  case "a":
  case "b":
    yield 1;
  default:
    yield 3;
};

// Switch Statement
switch (mode) {
  case "a":
  case "b":
    doSomething();
    break;
  default:
    doSomethingElse();
}

コンプライアンス・ソリューション

// Switch Expression
int i = switch (mode) {
  case "a", "b":
    yield 1;
  default:
    yield 3;
};

// Switch Statement
switch (mode) {
  case "a", "b":
    doSomething();
    break;
  default:
    doSomethingElse();
}

// Or even better:
switch (mode) {
  case "a", "b" -> doSomething();
  default -> doSomethingElse();
}
