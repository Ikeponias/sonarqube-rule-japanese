適切なインデントは、コードの読みやすさを向上させるシンプルで効果的な方法です。また、チーム内の開発者間でインデントを統一することで、ソースコントロールシステムにコミットされる差分を減らし、コードレビューを容易にすることができます。

このルールでは、インデントが設定値と一致しない場合に問題が発生します。インデントが悪いセクションの最初の行だけが報告されます。

コンプライアンス違反のコード例

インデントサイズが2の場合：

class Foo {
  public int a;
   public int b;   // Noncompliant, expected to start at column 4

...

  public void doSomething() {
    if(something) {
          doSomethingElse();  // Noncompliant, expected to start at column 6
  }   // Noncompliant, expected to start at column 4
  }
}

コンプライアンス・ソリューション

class Foo {
  public int a;
  public int b;

...

  public void doSomething() {
    if(something) {
        doSomethingElse();
    }
  }
}
