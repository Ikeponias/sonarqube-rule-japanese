できることだからといって、そうすべきとは限らないのが、三項演算の入れ子です。三項演算子をネストさせると、書いたときは明快でも、6ヶ月後にはメンテナ（あるいはもっと悪いことに、未来のあなた）が頭をかきむしりながら罵るようなコードになります。

その代わり、分かりやすさを重視して、入れ子になっている操作を別の文として表現するために、もう一行使ってください。

コンプライアンス違反のコード例

public String getReadableStatus(Job j) {
  return j.isRunning() ? "Running" : j.hasErrors() ? "Failed" : "Succeeded";  // Noncompliant
}

コンプライアンス・ソリューション

public String getReadableStatus(Job j) {
  if (j.isRunning()) {
    return "Running";
  }
  return j.hasErrors() ? "Failed" : "Succeeded";
}
