定義上、プリミティブ型はObjectsではないので、null にすることはできません。プリミティブ型に@CheckForNull や@Nullableをつけると、混乱が生じ、意味がありません。

このルールは、プリミティブ型（byte, short, int, long, float, double, boolean, char）を返すメソッドに@CheckForNull または@Nullable を設定した場合に問題が発生するものです。

コンプライアンス違反のコード例

@CheckForNull
boolean isFoo() {
 ...
}

コンプライアンス・ソリューション

boolean isFoo() {
 ...
}
