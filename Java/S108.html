多くの場合、コードのブロックは、コードの一部が本当に欠けているときに空っぽになります。そのため、このような空のブロックは、埋めるか削除する必要があります。

コンプライアンス違反のコード例

for (int i = 0; i < 42; i++){}  // Empty on purpose or missing piece of code ?

例外あり

ブロックにコメントが含まれている場合、synchronized のブロックでない限り、このブロックは空であるとはみなされません。synchronized ブロックがコメントを含んでいても、プログラムの流れに影響を与える可能性があるため、コメントはあっても空とみなされます。
