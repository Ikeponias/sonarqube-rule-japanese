StringBuilder やStringBuffer を文字でインスタンス化するのは誤解を招きやすい。なぜなら、ほとんどのJava開発者は、文字がStringBuffer の初期値になることを期待するからだ。

実際に起こるのは、文字のint表現を使って、StringBuffer の初期サイズを決定することです。

コンプライアンス違反のコード例

StringBuffer foo = new StringBuffer('x');   //equivalent to StringBuffer foo = new StringBuffer(120);

コンプライアンス・ソリューション

StringBuffer foo = new StringBuffer("x");
