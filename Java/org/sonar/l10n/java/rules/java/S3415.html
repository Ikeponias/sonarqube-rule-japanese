org.junit.Assert.assertEquals 、org.junit.Assert.assertSame などの標準的なアサーション・ライブラリ・メソッドは、第1引数を期待値、第2引数を実際の値と想定しています。AssertJの場合はその逆で、org.assertj.core.api.Assertions.assertThat の引数が実際の値で、それ以降の呼び出しに期待値が含まれます。これらを入れ替えても、テストは同じ結果（成功/失敗を繰り返す）になりますが、エラーメッセージは混乱することになります。

このルールは、アサーション・ライブラリ・メソッドの実際の引数がハードコードされた値で、期待される引数がそうでない場合に問題となる。

対応するフレームワーク

  • ジュウニットフォー
  • ジュウニットファイブ
  • アサートジェイ

コンプライアンス違反のコード例

org.junit.Assert.assertEquals(runner.exitCode(), 0, "Unexpected exit code");  // Noncompliant; Yields error message like: Expected:<-1>. Actual:<0>.
org.assertj.core.api.Assertions.assertThat(0).isEqualTo(runner.exitCode()); // Noncompliant

コンプライアンス・ソリューション

org.junit.Assert.assertEquals(0, runner.exitCode(), "Unexpected exit code");
org.assertj.core.api.Assertions.assertThat(runner.exitCode()).isEqualTo(0);
