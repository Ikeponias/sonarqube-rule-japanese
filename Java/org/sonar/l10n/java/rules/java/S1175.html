Object.finalize() は、オブジェクトが参照されなくなった後のある時点で、ガベージコレクタによって呼び出されます。

一般的に、Object.finalize() のオーバーロードは良くないとされています：

  • オーバーロードは、ガベージコレクタから呼び出されない場合があります。
  • ユーザーはObject.finalize() に電話することを想定していないので、混乱することになる。

しかし、それ以上に、実際にObject.finalize() をオーバーライドしていないメソッドに "finalize" という名前をつけるのは、ひどいアイデアです。

コンプライアンス違反のコード例

public int finalize(int someParameter) {        // Noncompliant
  /* ... */
}

コンプライアンス・ソリューション

public int someBetterName(int someParameter) {  // Compliant
  /* ... */
}
