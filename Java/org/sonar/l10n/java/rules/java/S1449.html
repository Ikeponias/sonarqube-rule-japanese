String オブジェクトのメソッドtoLowerCase() 、toUpperCase() 、format() を呼び出す際にロケールを指定しないと、システムのデフォルトエンコーディングが使用され、国際文字や数値表現で問題が発生する可能性があります。例えばトルコ語では、小さな文字「i」を大文字に変換すると、大文字の「I」の上にドットが付いたものになります。

ロケールなしの大文字小文字変換は、「ホーム」環境では問題なく動作しても、異なるエンコーディングを使用する顧客にとっては診断が非常に困難な方法で壊れることがあります。このようなバグは、完全にではないにせよ、いざ修正しようと思っても再現が不可能な場合があります。ロケールに依存する文字列には、常に正しいロケールを使用すべきですが、大文字小文字を区別しないものにはLocale.ROOT を使用することができます。

コンプライアンス違反のコード例

myString.toLowerCase()

コンプライアンス・ソリューション

myString.toLowerCase(Locale.TR)
