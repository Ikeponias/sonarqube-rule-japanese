外部スコープで宣言された変数のオーバーライドやシャドウイングは、コードの可読性、ひいては保守性に強く影響します。さらに、ある変数を使用しているつもりが、実際には別の変数を使用していたために、保守者がバグを発生させる可能性もあります。

コンプライアンス違反のコード例

class Foo {
  public int myField;

  public void doSomething() {
    int myField = 0;
    ...
  }
}

見る

  • CERT、DCL01-C。- サブスコープで変数名を再利用しない
  • CERT、DCL51-J。- サブスコープ内の識別子を影で隠したり、不明瞭にしないこと
