演算子のペア（=+,=- or=! ）を使用した場合、逆に単一の演算子（+=,-= or!= ）を使用した場合は、コンパイルと実行はできますが、期待した結果は得られません。

このルールは、=+ 、=- 、=! が、2つの演算子の間にスペースがなく、かつ、その後に少なくとも1つの空白文字がある場合に問題が発生することを示しています。

コンプライアンス違反のコード例

int target = -5;
int num = 3;

target =- num;  // Noncompliant; target = -3. Is that really what's meant?
target =+ num; // Noncompliant; target = 3

コンプライアンス・ソリューション

int target = -5;
int num = 3;

target = -num;  // Compliant; intent to assign inverse value of num is clear
target += num;
