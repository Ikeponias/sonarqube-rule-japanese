Java APIの初期のクラス、例えばVector 、Hashtable 、StringBuffer などは、スレッドセーフにするために同期化されています。残念ながら、これらのコレクションをシングルスレッドから使用する場合でも、同期は性能に大きな悪影響を及ぼします。

同期していない新しい代替品を使用するのがよいでしょう：

  • ArrayList または、Vector の代わりにLinkedList
  • Deque ではなく、Stack 。
  • HashMap ではなく、Hashtable 。
  • StringBuilder ではなく、StringBuffer 。

同期コンテキストで使用される場合でも、その使用方法は巧妙であるため、使用する前によく考える必要があります。もし、その使い方が正当なものであると確信しているのであれば、この警告を無視していただいて結構です。

コンプライアンス違反のコード例

Vector cats = new Vector();

コンプライアンス・ソリューション

ArrayList cats = new ArrayList();

例外あり

これらの同期化されたクラスの使用は、オーバーライドするメソッドのシグネチャでは無視されます。

@Override
public Vector getCats() {...}
