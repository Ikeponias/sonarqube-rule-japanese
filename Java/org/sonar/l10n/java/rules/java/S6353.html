正規表現の構文では、同じことを何通りにも表現することが可能です。例えば、2桁の数字にマッチさせる場合、[0-9]{2,2} や\d{2} と書くことができます。この規則では、かさばる量化子や文字クラスを、より簡潔な同等物に置き換えることを推奨しています：

  • \d for[0-9] and\D for[^0-9]
  • \w for[A-Za-z0-9_] and\W for[^A-Za-z0-9_]
  • . 全てにマッチする文字クラス（例：[\w\W],[\d\D], または[\s\S] とs フラグ）の場合
  • x? forx{0,1},x* forx{0,},x+ forx{1,},x{N} forx{N,N}

コンプライアンス違反のコード例

"[0-9]" // Noncompliant - same as "\\d"
"[^0-9]" // Noncompliant - same as "\\D"
"[A-Za-z0-9_]" // Noncompliant - same as "\\w"
"[\\w\\W]" // Noncompliant - same as "."
"a{0,}" // Noncompliant - same as "a*"
