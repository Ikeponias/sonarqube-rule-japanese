例外は、エラーが発生した時点のアプリケーションの状態を表すものである。

Exception クラスの全フィールドをfinal にすることで、この状態を確実にします：

  • Exception がインスタンス化されると同時に完全に定義されます。
  • 疑わしいエラーハンドラによって更新されたり、破損したりしない。

これにより、開発者は何が問題だったのかを素早く理解することができるようになります。

コンプライアンス違反のコード例

public class MyException extends Exception {

  private int status;                               // Noncompliant

  public MyException(String message) {
    super(message);
  }

  public int getStatus() {
    return status;
  }

  public void setStatus(int status) {
    this.status = status;
  }

}

コンプライアンス・ソリューション

public class MyException extends Exception {

  private final int status;

  public MyException(String message, int status) {
    super(message);
    this.status = status;
  }

  public int getStatus() {
    return status;
  }

}
