ユーティリティクラスは、static メンバーのコレクションであり、インスタンス化されることを意図していません。拡張可能な抽象ユーティリティクラスであっても、パブリックコンストラクタを持つべきではありません。

Javaでは、少なくとも1つの明示的なコンストラクタを定義していないクラスには、暗黙のパブリックコンストラクタが追加されます。したがって、少なくとも1つの非パブリックコンストラクタを定義する必要があります。

コンプライアンス違反のコード例

class StringUtils { // Noncompliant

  public static String concatenate(String s1, String s2) {
    return s1 + s2;
  }

}

コンプライアンス・ソリューション

class StringUtils { // Compliant

  private StringUtils() {
    throw new IllegalStateException("Utility class");
  }

  public static String concatenate(String s1, String s2) {
    return s1 + s2;
  }

}

例外あり

クラスがpublic static void main(String[] args) メソッドを含む場合、ユーティリティクラスとはみなされず、本ルールでは無視されます。
