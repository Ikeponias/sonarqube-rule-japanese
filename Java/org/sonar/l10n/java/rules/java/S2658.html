動的にロードされたクラスは、静的なクラスイニシャライザによって実行される悪意のあるコードを含む可能性があります。つまり、そのようなクラスのメソッドをインスタンス化したり、明示的に呼び出したりしなくても、攻撃を受ける可能性があるのです。

このルールは、ダイナミック・クラス・ローディングの使用ごとに問題が発生します。

コンプライアンス違反のコード例

String className = System.getProperty("messageClassName");
Class clazz = Class.forName(className);  // Noncompliant
