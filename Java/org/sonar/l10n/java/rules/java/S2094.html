空のクラスには良い言い訳はありません。もし、単に共通の拡張ポイントとして使われているのであれば、interface に置き換えるべきでしょう。また、将来の開発のためのプレースホルダとして挿入されたのであれば、その内容を充実させる必要があります。それ以外の場合は、削除されるべきです。

コンプライアンス違反のコード例

public class Nothing {  // Noncompliant
}

コンプライアンス・ソリューション

public interface Nothing {
}

例外あり

空のクラスはマーカータイプとして使用できるため（例えばSpringの場合）、アノテーションが付けられた空のクラスは無視されます。

@Configuration
@EnableWebMvc
public final class ApplicationConfiguration {

}
