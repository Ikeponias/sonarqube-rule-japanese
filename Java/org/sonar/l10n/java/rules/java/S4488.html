Spring framework 4.3では、アノテーションされたメソッドのセマンティクスをよりよく表現するために、@RequestMapping アノテーションのバリエーションを導入しました。@GetMapping,@PostMapping,@PutMapping,@PatchMapping and@DeleteMappingの使用は、生の@RequestMapping(method = RequestMethod.XYZ) の使用よりも優先されるべきです。

コンプライアンス違反のコード例

@RequestMapping(path = "/greeting", method = RequestMethod.GET) // Noncompliant
public Greeting greeting(@RequestParam(value = "name", defaultValue = "World") String name) {
...
}

コンプライアンス・ソリューション

@GetMapping(path = "/greeting") // Compliant
public Greeting greeting(@RequestParam(value = "name", defaultValue = "World") String name) {
...
}
