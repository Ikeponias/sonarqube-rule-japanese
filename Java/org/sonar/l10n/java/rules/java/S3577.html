命名規則を共有することで、チームは効率的にコラボレーションを行うことができます。このルールでは、テストクラス名が提供された正規表現に一致しない場合に問題が発生します。

コンプライアンス違反のコード例

デフォルト値で：^((Test|IT)[a-zA-Z0-9_]+|[A-Z][a-zA-Z0-9_]*(Test|Tests|TestCase|IT|ITCase))$

class Foo {  // Noncompliant
  @Test
  void check() {  }
}

class Bar {  // Noncompliant
  @Nested
  class PositiveCase {
    @Test
    void check() {  }
  }
}

コンプライアンス・ソリューション

class FooTest {
  @Test
  void check() {  }
}

class BarIT {
  @Nested
  class PositiveCase {
    @Test
    void check() {  }
  }
}
