NullPointerException は、null の値が予期せず発生したことを示す必要があります。良いプログラミングの実践では、NPEを回避するためにコードを構造化することを指示します。

明示的にNullPointerException を投げることは、それを避けるためのコーディングではなく、メソッドの呼び出し元が明示的にそれをキャッチすることを強制します。さらに、予期せず発生したnull の値と、メソッドが意図的に NPE をスローする原因となる条件とを区別することが難しくなります。

メソッドのパラメータがNULLであってはならないことを示すNPEが投げられる場合、代わりに@NotNull アノテーションを使用してください。

コンプライアンス違反のコード例

public void doSomething (String aString) throws NullPointerException {
     throw new NullPointerException();
}

コンプライアンス・ソリューション

public void doSomething (@NotNull String aString) {
}
