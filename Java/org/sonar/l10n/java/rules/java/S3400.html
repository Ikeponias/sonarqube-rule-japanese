常に同じ定数値を返すメソッドに、メソッド呼び出しのオーバーヘッドを強いる意味はない。さらに悪いことに、メソッド呼び出しが必要であるという事実は、そのメソッドを呼び出した開発者を、何かもっと多くのことが行われていると誤解させる可能性があります。代わりに定数を宣言してください。

このルールは、定数値（return ）のみのステートメントを含むメソッドで、問題が発生します。

コンプライアンス違反のコード例

int getBestNumber() {
  return 12;  // Noncompliant
}

コンプライアンス・ソリューション

static final int BEST_NUMBER = 12;

例外あり

@Override や Spring の@RequestMapping などのアノテーションを持つメソッドは無視されます。
