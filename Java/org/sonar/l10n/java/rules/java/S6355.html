Java 9以降、@Deprecated 、アノテーションに2つの引数が追加されました：

  • since は、非推奨がいつ行われたかを記述することができます。
  • forRemoval非推奨の要素が将来的に削除されるかどうかを示します。

メンテナの作業を楽にするために、これらの引数を常に1つまたは両方追加することが推奨されます。

このルールは、@Deprecated を引数なしで使用した場合の問題を報告するものです。

コンプライアンス違反のコード例

@Deprecated

コンプライアンス・ソリューション

@Deprecated(since="4.2", forRemoval=true)

例外あり

非推奨のクラスやインターフェースのメンバやメソッドは、このルールでは無視されます。クラスやインターフェイスそのものは、依然としてこの規則の対象となる。
