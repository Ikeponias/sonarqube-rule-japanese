長いパラメータリストは、多数のパラメータをラップするために新しい構造を作成する必要があるか、関数があまりにも多くのことを行っていることを示すことができます。

コンプライアンス違反のコード例

最大4つのパラメータを持つ：

public void doSomething(int param1, int param2, int param3, String param4, long param5) {
...
}

コンプライアンス・ソリューション

public void doSomething(int param1, int param2, int param3, String param4) {
...
}

例外あり

で注釈されたメソッド：

  • 春の@RequestMapping (および@GetRequest のような関連するショートカットアノテーション)
  • JAX-RS APIアノテーション（@javax.ws.rs.GET のようなものです。）
  • @org.springframework.beans.factory.annotation.Autowired による Bean コンストラクタ注入
  • @javax.inject.Inject による CDI コンストラクタ注入
  • @com.fasterxml.jackson.annotation.JsonCreator
  • マイクロノート社のアノテーション（@io.micronaut.http.annotation.Get のようなものです。）

は、多くのパラメータを持ち、カプセル化が可能である。そのようなメソッドは無視されます。

また、Springコンポーネントとしてアノテーションされたクラス（@org.springframework.stereotype.Component ）に単一のコンストラクタがある場合、そのコンストラクタは@Autowired とみなされ、ルール上無視されます。
