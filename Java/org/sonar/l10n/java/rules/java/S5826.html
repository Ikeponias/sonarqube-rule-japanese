JUnit3からJUnit4やJUnit5に移行する際に同じ動作を維持するために、setUp() とtearDown() メソッド（各テストの前後にコードのブロックを実行するためにJUnit3で最初に導入されました）は、同等のアノテーションを正しく付与する必要があります。

このルールにより、テストクラスでアノテーションされていないsetUp() とtearDown() のメソッドに問題が発生します。

コンプライアンス違反のコード例

  • JUnit4です： 
public void setUp() { ... } // Noncompliant; should be annotated with @Before
public void tearDown() { ... }  // Noncompliant; should be annotated with @After
  • JUnit5です： 
public void setUp() { ... } // Noncompliant; should be annotated with @BeforeEach
public void tearDown() { ... }  // Noncompliant; should be annotated with @AfterEach

コンプライアンス・ソリューション

  • JUnit4です： 
@Before
public void setUp() { ... }

@After
public void tearDown() { ... }
  • JUnit5です： 
@BeforeEach
void setUp() { ... }

@AfterEach
void tearDown() { ... }
