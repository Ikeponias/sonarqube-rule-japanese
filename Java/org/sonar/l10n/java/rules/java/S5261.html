else のぶら下がり問題は、入れ子になったif/else の文が中括弧なしで書かれている場合に発生します。この場合、else は最も近いif に関連付けられますが、これは必ずしも明らかではなく、インデントが誤解を招くこともあります。

このルールでは、else ステートメントが中括弧なしでネストされたif ステートメントの中にあるため、理解するのが困難であることを報告します。

中括弧をつけると、一般的にコードが明確になります（ルール{rule:java:S121}参照）が、このダングリングelse の状況では、コードの意図を本当に明確にすることができます。

コンプライアンス違反のコード例

 if (a)
   if (b)
     d++;
 else     // Noncompliant, is the "else" associated with "if(a)" or "if (b)"? (the answer is "if(b)")
   e++;

コンプライアンス・ソリューション

 if (a) {
   if (b) {
     d++;
   }
 } else { // Compliant, there is no doubt the "else" is associated with "if(a)"
   e++;
 }
