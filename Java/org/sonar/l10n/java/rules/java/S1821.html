入れ子のswitch 構造は、内側のswitch のケースを外側のステートメントや式に属するものと混同しやすいため、理解しにくい。したがって、switch ステートメントや式が入れ子になっているのは避けるべきです。

具体的には、switch 文や式を入れ子にする必要がないようにコードを構成する必要がありますが、それができない場合は、内側のswitch を別のメソッドに移動することを検討してください。

コンプライアンス違反のコード例

void foo(int n, int m) {
  switch (n) {
    case 0:
      switch (m) {  // Noncompliant; nested switch
        // ...
      }
    case 1:
      // ...
    default:
      // ...
  }
}

コンプライアンス・ソリューション

void foo(int n, int m) {
  switch (n) {
    case 0:
      bar(m);
    case 1:
      // ...
    default:
      // ...
  }
}

void bar(int m){
  switch(m) {
    // ...
  }
}
