アサーションは、テストコードで使用することを意図していますが、プロダクションコードでは使用しません。ビルドツールがテストスコープで必要な依存関係しか提供しない場合、混乱しますし、ClassNotFoundException につながるかもしれません。

また、アサーションはError のサブクラスを投げることになります：AssertionErrorのサブクラスを投げます。

このルールは、テストで使用されることを意図したアサーションが、プロダクションコードで使用される場合に問題となる。

対応するフレームワーク

  • ジュニト
  • フェスタアサート
  • アサートジェイ

注：Java自体からのassert 、またはソースコードのパッケージ名がテストに関連している場合（含む：test またはassert またはjunit ）は、この限りではありません。
