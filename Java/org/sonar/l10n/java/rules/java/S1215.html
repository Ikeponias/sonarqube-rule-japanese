System.gc() やRuntime.getRuntime().gc() を呼び出すのは、単純な理由で良くない。JVMのベンダー、バージョン、オプションによって動作が変わるため、JVMの内部で何が行われるかを正確に知る術がない：

  • 通話中にアプリケーション全体がフリーズしてしまうのでしょうか？
  • -XX:DisableExplicitGC オプションは有効になっていますか？
  • JVMは単に呼び出しを無視するのでしょうか？
  • ...

System.gc() と同様に、最終化を保留しているオブジェクトの最終化メソッドを強制的に呼び出すために、runFinalization() を手動で呼び出す理由はない。

これらの予測不可能なメソッドに依存するアプリケーションも予測不可能であるため、壊れてしまいます。ガベージコレクタを実行し、finalize() メソッドを呼び出すタスクは、JVM にのみ委ねられるべきです。
