マジックナンバーとは、どこからともなく出てくる数字で、文の中で直接使われるものです。マジックナンバーは、例えばループの反復回数を制限したり、プロパティの値をテストしたりするためによく使われます。

マジックナンバーの使用は、コードを書いているときには明白でわかりやすいと思われるかもしれませんが、デバッグのときにはもっと明白でわかりにくくなります。

そのため、マジックナンバーは、まず明確に名付けられた変数に割り当てられてから使用されるなど、神秘的なものでなければなりません。

-1、0、1 はマジックナンバーとみなされません。

コンプライアンス違反のコード例

public static void doSomething() {
	for(int i = 0; i < 4; i++){                 // Noncompliant, 4 is a magic number
		...
	}
}

コンプライアンス・ソリューション

public static final int NUMBER_OF_CYCLES = 4;
public static void doSomething() {
  for(int i = 0; i < NUMBER_OF_CYCLES ; i++){
    ...
  }
}

例外あり

このルールでは、hashCode 方式は無視されます。
