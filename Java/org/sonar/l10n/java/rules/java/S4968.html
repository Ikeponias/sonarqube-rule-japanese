型変数やワイルドカードがfinal である上限を宣言する場合，実行時に1つの型しか受け入れないため，パラメトリゼーションは全く汎用的ではありません：final です．Generics を使う代わりに，具象のfinal クラスを直接使うほうが単純です．

コンプライアンス違反のコード例

public static <T extends String> T getMyString() { // Noncompliant; String is a "final" class and so can't be extended
 [...]
}

コンプライアンス・ソリューション

public static String getMyString() { // Compliant
  [...]
}
