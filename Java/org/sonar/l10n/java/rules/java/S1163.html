最終ブロック内で例外をスローすると、try またはcatchブロック内で以前にスローされた例外がマスクされ、マスクされた例外メッセージとスタックトレースは失われます。

コンプライアンス違反のコード例

try {
  /* some work which end up throwing an exception */
  throw new IllegalArgumentException();
} finally {
  /* clean up */
  throw new RuntimeException();       // Noncompliant; masks the IllegalArgumentException
}

コンプライアンス・ソリューション

try {
  /* some work which end up throwing an exception */
  throw new IllegalArgumentException();
} finally {
  /* clean up */
}

見る

  • CERT、ERR05-J。- チェックされた例外を最終ブロックから逃がさないようにする
