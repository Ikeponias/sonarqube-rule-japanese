パブリッククラスの変数フィールドは、カプセル化の原則を尊重せず、主に3つのデメリットがあります：

  • バリデーションなどの動作を追加することはできません。
  • 内部表現が公開され、後から変更することはできません。
  • メンバーの値は、コードのどこからでも変更可能であり、プログラマーの想定に沿わない場合があります。

プライベート属性とアクセサーメソッド（set、get）を使用することで、不正な改変を防止することができます。

コンプライアンス違反のコード例

public class MyClass {

  public static final int SOME_CONSTANT = 0;     // Compliant - constants are not checked

  public String firstName;                       // Noncompliant

}

コンプライアンス・ソリューション

public class MyClass {

  public static final int SOME_CONSTANT = 0;     // Compliant - constants are not checked

  private String firstName;                      // Compliant

  public String getFirstName() {
    return firstName;
  }

  public void setFirstName(String firstName) {
    this.firstName = firstName;
  }

}

例外あり

それらは変更不可能であるため、このルールはpublic final フィールドを無視する。また、注釈付きフィールドも、注釈が何であれ無視されます。注釈はインジェクションフレームワークによってしばしば使用され、その代わりに公開フィールドを持つことが要求されるからです。
