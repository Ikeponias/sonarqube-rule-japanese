このルールは、if の記述の後に、1つ以上のelse if の記述がある場合に適用されます。最後のelse ifの後には、else の記述が必要です。

最終的にelse の記載が必要なのは、守備範囲の広いプログラミングです。

else ステートメントは、適切な行動をとるか、なぜ行動をとらないかについての適切なコメントを含むべきである。これは、switch ステートメントに最終的なdefault 節を設けるという要求と一致する。

コンプライアンス違反のコード例

if (x == 0) {
  doSomething();
} else if (x == 1) {
  doSomethingElse();
}

コンプライアンス・ソリューション

if (x == 0) {
  doSomething();
} else if (x == 1) {
  doSomethingElse();
} else {
  throw new IllegalStateException();
}
