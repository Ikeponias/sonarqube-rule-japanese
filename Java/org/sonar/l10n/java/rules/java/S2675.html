readObject メソッドは、Serializable オブジェクトがファイルから水分を補給するために特別な処理を必要とする場合に記述されます。readObject で生成されるオブジェクトは、このメソッドを呼び出したスレッドにしか見えないので、synchronized キーワードは不要であり、synchronized を使うのは混乱を招くだけである、ということになるはずです。もしそうでないなら、このメソッドをリファクタリングして、そうなるようにすべきです。

コンプライアンス違反のコード例

private synchronized void readObject(java.io.ObjectInputStream in)
     throws IOException, ClassNotFoundException { // Noncompliant
  //...
}

コンプライアンス・ソリューション

private void readObject(java.io.ObjectInputStream in)
     throws IOException, ClassNotFoundException { // Compliant
  //...
}
