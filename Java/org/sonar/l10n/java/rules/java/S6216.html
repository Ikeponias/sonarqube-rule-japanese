一般に、クラス、メソッド、フィールドのアクセシビリティを変更したり回避したりすると、カプセル化の原則に違反し、ランタイムエラーにつながる可能性があります。レコードの場合はさらに厄介で、レコードのフィールドの可視性を変更することはできず、既存の値を更新しようとすると、実行時にIllegalAccessException につながります。

このルールは、リフレクションがレコードのフィールドの可視性を変更するために使用される場合と、レコードのフィールドの値を直接更新するために使用される場合に問題が発生します。

コンプライアンス違反のコード例

record Person(String name, int age) {}

Person person = new Person("A", 26);
Field field = Person.class.getDeclaredField("name");
field.setAccessible(true); // secondary
field.set(person, "B"); // Noncompliant
