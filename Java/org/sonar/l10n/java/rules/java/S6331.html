正規表現でグループを使用する理由はいくつかあります：

  • 優先順位を変更する（例：do(g|or) は 'dog' と 'door' にマッチする）。
  • グループをキャプチャする場合、マッチの括弧付き部分を記憶します。
  • 読みやすさを向上させるために

いずれにせよ、空のグループを持つことは、おそらく間違いです。リファクタリング後の残骸で削除されるべきか、あるいは実際の括弧が意図されたもので、エスケープされなかったかのどちらかです。

コンプライアンス違反のコード例

"foo()"  // Noncompliant, will match only 'foo'

コンプライアンス・ソリューション

"foo\\(\\)"  // Matches 'foo()'
