技術の指紋を開示することで、攻撃者はウェブアプリケーションの開発に使用された技術に関する情報を収集し、関連するセキュリティ評価（既知の脆弱なコンポーネントの特定など）をより迅速に実行することができます。

ということを自問自答してください。

  • アプリケーションでは、x-powered-by HTTPヘッダーなどが使用されます。
  • アプリケーションで使用される技術は機密事項であり、容易に推測できないようにする必要があります。

これらの質問のいずれかに「はい」と答えた方はリスクがあります。

推奨されるセキュアコーディングの慣行

例えば、x-powered-by HTTPヘッダーなどで、ウェブサイトで使用されている技術を開示しないことが推奨されます。

また、このHTTPヘッダーはランダムな値を設定するのではなく、完全に無効化する方がよい。

センシティブコードの例

public ResponseEntity<String> testResponseEntity() {
  HttpHeaders responseHeaders = new HttpHeaders();
  responseHeaders.set("x-powered-by", "myproduct"); // Sensitive

  return new ResponseEntity<String>("foo", responseHeaders, HttpStatus.CREATED);
}

コンプライアンス・ソリューション

x-powered-by やServer HTTPヘッダーなど、アプリケーションのフィンガープリントを開示するような手段は使わないでください。
