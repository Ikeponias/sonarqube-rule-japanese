Serializable クラスにserialVersionUID フィールドを設けることは、Serializableのドキュメントで強く推奨されていますが、その推奨に盲従することは有害です。

serialVersionUID このフィールドはデータをデシリアライズするときに検証され、データを読むコードがシリアライズされたデータと互換性があることを確認します。失敗した場合は、シリアライズされたデータとコードが同期していないことを意味し、何が間違っているのかがわかるので、これは結構です。

IDEで生成されたserialVersionUID 、あるいはやみくもにハードコーディングした場合、Serializable クラスに追加のフィールドが追加されたときに、serialVersionUID の値を更新するのを忘れてしまう可能性が高くなります。その結果、古いシリアライズデータが新しいバージョンのコードと互換性があると誤認され、デバッグが困難な状況が発生します。

したがって、serialVersionUID の定義は慎重に行う必要がある。このルールは、Serializable を実装するクラスで宣言された各serialVersionUID フィールドについて、serialVersionUID フィールドの存在と値がチームによって挑戦され、検証されることを確認するために問題を提起します。

コンプライアンス違反のコード例

public class Foo implements Serializable {
  private static final long serialVersionUID = 1;
}

public class BarException extends RuntimeException {
  private static final long serialVersionUID = 8582433437601788991L;
}
