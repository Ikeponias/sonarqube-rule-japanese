クラスメンバが、その囲むクラスと同じ名前（大文字小文字の違いは別として）であることは、混乱を招く。特に、クラスインスタンスにクラス自体の名前を付けるという一般的な慣習を考えると、そのようなことが起こり得ます。

ベストプラクティスでは、クラスと同じ名前のフィールドやメンバは、そのクラスが表す、または保持する特定の側面をより説明するために名前を変更することが指示されています。

コンプライアンス違反のコード例

public class Foo {
  private String foo;

  public String getFoo() { }
}

Foo foo = new Foo();
foo.getFoo() // what does this return?

コンプライアンス・ソリューション

public class Foo {
  private String name;

  public String getName() { }
}

//...

Foo foo = new Foo();
foo.getName()

例外あり

フィールドの型が包含クラスであり、そのフィールドが静的である場合、型のような名前のシングルトンを許可する問題は発生しない。

public class Foo {
  ...
  private static Foo foo;
  public Foo getInstance() {
    if(foo==null) {
      foo = new Foo();
    }
    return foo;
  }
  ...
}
