ゲッターとセッターは、private フィールドへの制御されたアクセスを与えるpublic メソッドを提供することで、カプセル化を強制する方法を提供します。しかし、複数のフィールドを持つクラスでは、必要なゲッターやセッターを素早く作成するためにコピー＆ペーストが使われることが珍しくありません。その結果、ゲッターやセッターによって間違ったフィールドがアクセスされることがあります。

このルールは、いずれの場合にも問題を提起するものである：

  • セッターは、対応する名前のフィールドを更新するわけではありません。
  • ゲッターは、対応する名前のフィールドにアクセスしない。

コンプライアンス違反のコード例

class A {
  private int x;
  private int y;

  public void setX(int val) { // Noncompliant: field 'x' is not updated
    this.y = val;
  }

  public int getY() { // Noncompliant: field 'y' is not used in the return value
    return this.x;
  }
}

コンプライアンス・ソリューション

class A {
  private int x;
  private int y;

  public void setX(int val) {
    this.x = val;
  }

  public int getY() {
    return this.y;
  }
}
