メソッドがメソッド本体を持たない理由はいくつかあります：

  • これは意図しない省略であり、本番での予期せぬ動作を防ぐために修正する必要があります。
  • まだサポートされていない、あるいは今後もサポートされることはない。この場合、UnsupportedOperationException を投げる必要があります。
  • このメソッドは、意図的に空白のオーバーライドを行うものです。この場合、ネストされたコメントで空白のオーバーライドの理由を説明する必要があります。

コンプライアンス違反のコード例

public void doSomething() {
}

public void doSomethingElse() {
}

コンプライアンス・ソリューション

@Override
public void doSomething() {
  // Do nothing because of X and Y.
}

@Override
public void doSomethingElse() {
  throw new UnsupportedOperationException();
}

例外あり

なお、以下の場合は問題ありません：

  • 非公開のデフォルト（引数なし）コンストラクタ
  • クラス内に他のコンストラクタがある場合、デフォルト（引数なし）のコンストラクタを公開する。
  • 抽象クラスにおける空のメソッド
  • @org.aspectj.lang.annotation.Pointcut() でアノテーションされたメソッド 
public abstract class Animal {
  void speak() {  // default implementation ignored
  }
}
