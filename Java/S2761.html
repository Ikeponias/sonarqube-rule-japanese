演算子の無駄な繰り返しは、たいていタイプミスです。!i で済むところを!!!i と書く理由はない。

一方、インクリメントとデクリメントという演算子の繰り返しは、意図的にやっているのかもしれませんが、そうすることで意味がわからなくなるので、簡略化すべきです。

このルールは、以下のようなシーケンスで問題が発生します：! ~,-, および+ 。

コンプライアンス違反のコード例

int i = 1;

int j = - - -i;  // Noncompliant; just use -i
int k = ~~~i;    // Noncompliant; same as i
int m = + +i;    // Noncompliant; operators are useless here

boolean b = false;
boolean c = !!!b;   // Noncompliant

コンプライアンス・ソリューション

int i =  1;

int j = -i;
int k = ~i;
int m =  i;

boolean b = false;
boolean c = !b;

例外あり

~~ を使用した GWT コンパイル時のオーバーフロー処理は無視されます。
