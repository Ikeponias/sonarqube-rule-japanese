このルールは、Collection の項目に対する反復処理を、Collection が扱う型のスーパータイプに対して行う場合に問題となる。

Object やObject とCollection で扱われる実際のクラスとの間のクラスに依存することは推奨されません。言語としては認められていますが、このやり方はコードの可読性を低下させ、Collection のアイテムをダウンキャストしてメソッドを呼び出すことを余儀なくされます。一方、繰り返しで正しい型を使用するだけで、物事はより明確でシンプルになります。

コンプライアンス違反のコード例

public Collection<Person> getPersons() { ... }

for (Object item : getPersons()) { // Noncompliant
  Person person = (Person) item; // Noncompliant; it's required to down-cast to the to correct type to use "item"
  person.getAdress();
}

コンプライアンス・ソリューション

for (Person person : getPersons()) { // Compliant
  person.getAddress() ;
}
