一般的なグッドプラクティスは、1つの論理的概念のみを対象とし、1つの理由でのみ失敗するテストメソッドを書くことです。

1つの概念をテストするために複数のアサーションを持つことは理にかなっているかもしれませんが、あまりにも多くのアサーションを持つことは、テストが複雑になりすぎて、複数のアサーションにリファクタリングする必要があることを意味します。

このルールは、所定の数以上のアサーションを含むテストメソッドを報告します。

コンプライアンス違反のコード例

2.のパラメータを持つ。

@Test
void test() { // Refactor this method.
  assertEquals(1, f(1));
  assertEquals(2, f(2));
  assertEquals(3, g(1));
}

コンプライアンス・ソリューション

void test_f() {
  assertEquals(1, f(1));
  assertEquals(2, f(2));
}
void test_g() {
  assertEquals(3, g(1));
}
