Java 10では、Local-Variable Type Inferenceが導入されました。これは、var キーワードで宣言することで、変数の期待される型を省略することができる。

この新しい変数宣言の方法を使うことが常に可能であり、きれいであるとは言えませんが、左側の型と右側の型が割り当てで同じである場合、var を使用すると、より簡潔なコードになります。

このルールは、変数の期待される型が代入された式の返される型と同じで、その型が読者によって容易に推測できる場合（名前またはイニシャライザで型がすでに言及されている場合、または式が自明である場合）に問題を報告します。

コンプライアンス違反のコード例

MyClass myClass = new MyClass();

int i = 10; // Type is self-explanatory

MyClass something = MyClass.getMyClass(); // Type is already mentioned in the initializer

MyClass myClass = get(); // Type is already mentioned in the name

コンプライアンス・ソリューション

var myClass = new MyClass();

var i = 10;

var something = MyClass.getMyClass();

var myClass = get();
