if 文やfor ループなどの1行ブロックでは、中括弧を省略することができますが、そうすることで誤解を招いたり、バグを誘発することがあります。

このルールは、1行ブロックの後の行の空白が、その行をブロックに含める意図を示すが、中括弧が省略されているため、その行が無条件に1回実行されることを意味する場合に問題が発生する。

なお、このルールでは、タブ文字は1スペースと同等とみなしています。スペースとタブを混在させると、エディタ上では問題ないが、タブの大きさを変えたときに混乱する問題がコードに発生することがあります。

コンプライアンス違反のコード例

if (condition)
  firstActionInBlock();
  secondAction();  // Noncompliant; executed unconditionally
thirdAction();

if (condition) firstActionInBlock(); secondAction();  // Noncompliant; secondAction executed unconditionally

if (condition) firstActionInBlock();  // Noncompliant
  secondAction();  // Executed unconditionally

if (condition); secondAction();  // Noncompliant; secondAction executed unconditionally

String str = null;
for (int i = 0; i < array.length; i++)
  str = array[i];
  doTheThing(str);  // Noncompliant; executed only on last array element

コンプライアンス・ソリューション

if (condition) {
  firstActionInBlock();
  secondAction();
}
thirdAction();

String str = null;
for (int i = 0; i < array.length; i++) {
  str = array[i];
  doTheThing(str);
}
