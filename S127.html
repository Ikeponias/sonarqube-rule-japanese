for ループ停止条件は、ループカウンターを不変の値（すなわち、ループの繰り返しの開始時と終了時の両方で真である値）に対してテストする必要があります。理想的には、ループが始まる直前に停止条件がローカル変数に設定されることを意味します。

不変でない停止条件は、効率が若干悪いだけでなく、理解やメンテナンスが難しく、将来的にエラーの混入につながる可能性が高いです。

このルールは、3種類の非変動停止状態を追跡します：

  • for ループ本体でループカウンターが更新された場合
  • 停止条件がメソッド呼び出しに依存する場合
  • 停止条件がオブジェクトのプロパティに依存する場合、そのプロパティはループの実行中に変更される可能性があるためです。

コンプライアンス違反のコード例

for (int i = 0; i < 10; i++) {
  ...
  i = i - 1; // Noncompliant; counter updated in the body of the loop
  ...
}

コンプライアンス・ソリューション

for (int i = 0; i < 10; i++) {...}
