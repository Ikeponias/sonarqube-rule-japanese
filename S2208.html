パッケージ内のすべてのクラスをやみくもにインポートすると、クラスの名前空間が乱れ、異なるパッケージ内の同じ名前のクラス間で衝突が起こる可能性があります。一方、必要なクラスを具体的にリストアップすることで、そのような問題を回避し、どのバージョンが求められていたかを明確にすることができます。

コンプライアンス違反のコード例

import java.sql.*; // Noncompliant
import java.util.*; // Noncompliant

private Date date; // Date class exists in java.sql and java.util. Which one is this?

コンプライアンス・ソリューション

import java.sql.Date;
import java.util.List;
import java.util.ArrayList;

private Date date;

例外あり

静的インポートはこのルールで無視されます。E.G.

import static java.lang.Math.*;
